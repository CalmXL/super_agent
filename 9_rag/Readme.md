# RAG 实战

Retrieval-Augmented Generation

先检索，再生成

## RAG 的六个步骤

1. 加载
    把文档读进来。md、纯文本直接读，pdf 则需要解析工具（pdf-parse等）。
2. 分块
    把长文档切成小段。分块的目标是让每个小段包含一个完整的语义单元。
3. 向量化
    把每个小段转成一串数字（emebeding 向量）。语义相近的文本在向量空间里距离更近。
4. 存储
    把向量和原文一起存下来。SQLite + sqlite-vec(向量索引) + FTS5 (全文索引)
5. 检索
    把用户提问，把问题也向量化，然后再存储中找最相近的片段。我们使用 混合检索。
6. 注入
    把检索到的片段塞进上下文，让模型基于真实内容生成问答。
